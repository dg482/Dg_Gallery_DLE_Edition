Режим "Портал" - user_mode1.tpl

 

В режиме «Портал» пользователь может создавать альбомы размещая их в доступных категориях. В данном режиме в главном окне пользователю доступны следующие действия:

1.Просмотр своих альбомов.
2.Создание альбома.
3.Поиск фаилов используя фильтр.

Для ознакомления пользователей с возможностями скрипта рекомендуется добавить дополнительную вкладку в главное окно и разместить в ней необходимую информацию, также в данном окне Вы можете разметсить рекламные материалы.  Добавление вкладок осуществляется в ручном режиме, отвечающий за первую вкладку код отмечен подчеркнутым шрифтом. Для упрощения сложных конструкций в шаблоне  user_mode1.tpl поддерживается включение шаблонов. Часть HTML структуры пользовательского интерфейса  размещена в php (/app/view/user.php).

За редактирование альбомов отвечает шаблон edit_albom.tpl

Вы можете в произвольном порядке менять оформление как отдельных элементов так и всего пользовательского интерфейса. При редактирование соблюдайте существующую структуру HTML, не изменяйте идентификаторы и атрибуты элементов форм.

1.[list-albom][/list-albom] – дублируемый текст, список альбомов
2.[link][/link] – ссылка на альбом применяетя в контексе тега [list-albom][/list-albom].
3.{cover} – ссылка на обложку или случайный файл из альбома, применяетя в контексе тега [list-albom][/list-albom].
4.{title} – название альбома, применяетя в контексе тега [list-albom][/list-albom].
5.[edit-link][/edit-link] – ссылка на редактирование альбома, применяетя в контексе тега [list-albom][/list-albom].
6.{albom} – HTML разметка для формы фильтров, список альбомов
7.[file-list][/file-list] – дублируемый текст, список файлов.
8.{id-file} – идентификатор файла, применяетя в контексе тега [file-list][/file-list]
9.{name} – имя файла на сервере, применяетя в контексе тега [file-list][/file-list]
10.{preview-path} – путь к превью изображения, применяетя в контексе тега [file-list][/file-list
